進階 Prompt Engineering 實戰：優化你的 AI 助手

企業內部 AI 輔助開發教育訓練

🎯 課程目標 Session Goals

將 Prompt Engineering 從「魔法」升級為工程學，精準操控 AI 的生成結果。

本堂課將帶領大家回歸根本，將 Prompt Engineering 視為意圖轉化指令的學問：

  • 精準控制： 擺脫空泛詞彙，透過嚴謹的語法與格式控制 AI。
  • 結構化指令： 運用 Markdown 的層級架構，讓 AI 清晰理解任務與邊界。
  • 架構思維： 將模糊的業務請求，轉化為模型可高效執行的精準約束。

📖 三大核心 意圖轉化約束

掌握三大高階技巧，構建具備工業級穩定度與高可預測性的 Prompt。

🤖 結構化 (Structured

使用 Markdown 標籤與 YAML/JSON 格式，將「任務目標」、「上下文」、「約束條件」明確分區，拉高 AI 的解析效率。

⚙️ 少量樣本 (Few-Shot

與其用文字費力解釋複雜規範，不如直接給出 1 到 2 個完美的黃金程式碼範例，供 AI 精準依循。

💡 思維鏈 (CoT - Chain of Thought

對於複雜業務邏輯，強制 AI 「先分析推演步驟，再輸出程式碼」，顯著降低邏輯謬誤。

🛠️ 實戰演練 購物車折扣邏輯

重構購物車折扣邏輯，將直覺式 Prompt 升級為結構化 + Few-Shot + CoT 組合技。

❌ 直覺型 Prompt：「幫我寫計算總價的 C#，VIP 打 8 折，滿千送百，處理一下精度。」（邏輯混亂且易遺漏精度）

✔️ 結構化 Prompt 重構：

example.md
# 任務：實作 IDiscountCalculator 介面
## 約束條件：
1. 金額計算必須使用 `decimal`，防止浮點數精度問題。
2. 傳入負數或零時，必須拋出 `ArgumentException` 例外。
## 少量樣本 (Few-Shot)：[在此提供 StandardCalculator 的完美實作]
## 思考步驟 (CoT)：
1. 分析傳入參數與折扣邊界條件。
2. 計算 VIP 會員折扣比例。
3. 套用滿千送百邏輯，並確保無小數點溢出。
4. 最終生成 C# 實作代碼。

⚠️ 常見誤區與 防禦機制

避開常見地雷，合理配置 AI 上下文，避免垃圾範本與無效算力浪費。

🤖 避免塞入整份規格書

資訊過載會使 AI 遺忘約束。應使用漸進式提示 (Progressive Prompting)，先要 Interface 與骨架，再填充實作。

⚙️ 杜絕劣質的 Few-Shot 範例

Few-Shot 具有極強的傳染性。範例程式碼必須是經過 Review 的「黃金標準」，若範例有 Bad Smells，AI 會全盤抄襲。

💡 切勿濫用 CoT

簡單任務（如格式轉換、直覺語法翻譯）開啟 CoT 只會浪費 Token 並拖慢速度；複雜業務邏輯與狀態機才需啟用 CoT。

❓ 常見問答 Q&A

探討 Prompt 在未來開發工具（如 Copilot）及日常工作流中的定位與固化方式。

❌ Q1: 既然有了自動完成的 Copilot，我們為什麼還要大費周章學習 Prompt Engineering？


A1: 定位完全不同。Copilot 是微觀層面的代碼補全，而進階的 Prompt Engineering 是宏觀層面的架構設計與系統邊界定義。懂 Prompt 的工程師才能主導大型模組的生成。

🎯 Q2: 每次寫這麼長的結構化 Prompt 真的很花時間，日常開發如何提高實用性？


A2: 固化為 Agent Skills！這就是第一堂課的核心價值。將寫好的結構化 Prompt 封裝成 Skill 卡匣，日常只需傳入變數，即可一鍵調用，打造你專屬的自動化代碼工廠。
