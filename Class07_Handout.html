第 7 堂：從對話到行動

Agent 的本質與運作機制總結

🎯 課程目標

  • 看透本質： 了解 Agent 與單純 Chatbot 的決定性差異。
  • 掌握 ReAct 框架： 學習 Agent 的「感知 - 思考 - 行動」迴圈。
  • 階段總結： 統整前 6 堂課的技術堆疊。

📖 什麼是 Agent？

Chatbot 只能說話，Agent 能夠行動。

  • Chatbot： 根據你的問題，預測下一段文字。
  • Agent (代理人)： 擁有目標、能使用工具 (Tools)、能觀察環境並自我修正。
💡 頓悟時刻 (Aha Moment)

Agent 不是魔法，而是一個 while(true) 迴圈。它不斷在：觀察 (Observation) ➔ 思考 (Thought) ➔ 行動 (Action)，直到達成目標才跳出迴圈！

🛠️ 核心機制：ReAct 框架

  • Reasoning (推理)： 分析當前狀況，決定下一步該怎麼做。
  • Acting (行動)： 呼叫外部工具（例如：讀取檔案、編譯程式、搜尋網路）。

實際範例：

思考： 我需要知道這個 function 怎麼用。
行動： grep_search("function_name")
觀察： 找到定義在 utils.cs 裡。
思考： 知道怎麼用了，開始寫 Code。

⚠️ 為什麼 Agent 會卡死？

  • 工具不穩定： 工具回傳了預期外的錯誤，Agent 陷入無限重試迴圈。
  • 目標模糊： 給出的指令太籠統，Agent 不知道何時算「完成」。
