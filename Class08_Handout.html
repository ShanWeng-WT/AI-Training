第 8 堂：制定專案的大腦

Context Files (AGENTS.md) 的威力

🎯 課程目標

  • 統一標準： 確保團隊中所有 AI 助手的行為與風格一致。
  • 建立專案大腦： 學會撰寫並維護全域上下文設定檔。
  • 區分 Context 類型： 理解靜態與動態上下文的差異。

📖 為什麼需要 AGENTS.md？

沒有全域規範的 AI 就像新來的外包人員，每次都要重新教專案架構。

  • 減少重複下指令： 把架構規範、命名慣例寫死在設定檔中。
  • 自動載入： AI 啟動時會自動閱讀，成為它的預設行為準則。
💡 頓悟時刻 (Aha Moment)

將開發規範寫進 AGENTS.md，等於把團隊的 Senior 工程師經驗「固化」並「複製」給每一個 AI Agent。你的 Agent 將比新進員工更懂專案規範！

🛠️ 實戰：靜態與動態 Context

  • 靜態 Context (AGENTS.md)：
    不會頻繁變動的架構規範 (例：Unity 只能用 ECS，不能寫 MonoBehaviour)。
  • 動態 Context (RAG)：
    隨時變動的實作細節 (例：某個 Weapon 腳本的當前狀態，讓 AI 自行去搜尋)。

# AGENTS.md 範例
## .NET Backend 規範
1. 所有的 API 必須回傳 `ApiResponse<T>`。
2. 嚴禁在 Controller 直接呼叫 DB，必須透過 Service 層。
3. Log 一律使用 Serilog，包含 CorrelationId。

⚠️ 常見誤區

  • 把設定檔當垃圾桶： 把所有細節都塞進 AGENTS.md，導致 Token 爆滿且 AI 抓不到重點。
  • 不更新： 專案架構改了，設定檔沒改，導致 AI 一直產出舊架構的 Code。
