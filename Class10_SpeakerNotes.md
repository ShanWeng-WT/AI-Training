# 第 10 堂：超越單次對話：Agentic Workflow 設計模式 - 講師講稿

**課程總時長：** 30 分鐘
**目標對象：** 希望進一步利用 AI 自動化高度複雜任務的開發者與系統工程師。
**主旨：** 深入探討吳恩達 (Andrew Ng) 提出的 Agentic Workflow，學習反思、工具使用、規劃與多代理協作。

---

## [00:00 - 05:00] 開場與 Agentic Workflow (Intro & Why)
- **(投影片：為何需要 Agentic Workflow？)**
- **講者台詞：**
  「歡迎來到第 10 堂課！你是不是常覺得 AI 寫複雜功能時總是出錯？因為傳統對話就像叫 AI 閉著眼睛寫幾百行 Code。
  今天我們要談的是『Agentic Workflow』。核心精神是讓 AI 像人類工程師一樣：能自己 Debug、能使用工具、會做計畫、會團隊合作。」

## [05:00 - 10:00] 模式一與二：反思與工具使用 (Reflection & Tool Use)
- **(投影片：Reflection & Tool Use)**
- **講者台詞：**
  「第一個模式是『反思』。我們寫個腳本，把 AI 產出的 Code 自動拿去編譯，如果噴錯，把錯誤訊息直接丟回給 AI 叫它修。這樣你最後拿到的就是能編譯的版本！
  第二個模式是『工具使用 (Function Calling)』。AI 不能通靈，如果你要它查玩家背包，你必須提供一個 API 工具給它。AI 分析語意後會呼叫你的 API，後端去查 DB，再把結果還給 AI。」

## [10:00 - 18:00] 模式三：規劃 (Planning)
- **(投影片：規劃模式 Planning)**
- **講者台詞：**
  「第三個模式：『規劃』。這是我認為最重要的。遇到龐大需求，一定要強迫 AI『先別寫程式！先給我步驟計畫！』
  我們把任務大卸八塊，然後透過自動化腳本，一次只叫它執行一個 Step。這樣能大幅減少幻覺，保持架構乾淨。」

## [18:00 - 24:00] 模式四：多代理協作 (Multi-agent Collaboration)
- **(投影片：多代理協作 Multi-agent)**
- **講者台詞：**
  「如果任務太複雜，一個 Agent 會搞混。我們就建一個虛擬團隊！
  一個 Coder Agent 專心寫程式；一個 Tech Lead Agent 專心做 Code Review。Coder 寫完，丟給 Tech Lead 檢查效能，如果有問題就退回要求重寫。這完美複製了人類開發管線。」

## [24:00 - 30:00] 防禦機制與結語 (Defenses & Q&A)
- **(投影片：常見陷阱與防禦機制)**
- **講者台詞：**
  「最後提醒幾個致命陷阱。給了 AI 自動權，要小心『無窮迴圈』，一定要設定 Max Iterations (最多重試 3 次)。
  另外，針對『危險操作』，像改 DB 或改系統檔案，絕對要加入 Human-in-the-loop，跳出按鈕讓你親自核准。
  今天的 4 個模式，回去可以立刻套用在你的自動化腳本裡。謝謝大家！」
