🎯 經典上標下內文標題 (以 `top-title-layout` 作為主要骨幹)

這是最主要的投影片架構，讀者由上而下閱讀，視覺動線流暢：

  • 優點一： 水平視覺延展寬廣，不容易造成偏向右側的視覺疲勞。
  • 優點二： 寬度充足，適合長句、多層次的清單條列。
  • 適用範圍： 90% 的常規教學投影片都可以以此為主要骨幹。
💡 小提示： 標題下方有一條全寬的漸層分割線，能清楚隔開標題與內文區域。

代碼優先：主控台雙欄排版 (以 `console-split` 呈現代碼)

左側可用來放置該程式碼的痛點或架構說明，右側則能直接展示對應的程式碼：

  • 沉浸式體驗： 帶有 macOS 風格的紅綠黃三點與半透明 IDE 邊框，極具工程質感。
  • 訊噪比極高： 將代碼置於專屬編輯器視窗，降低了與主畫面的文字混淆。
ExampleClass.cs 
public class ExampleClass
{
    // 右側最適合直接展示 C# 代碼或 YAML 配置檔
    public void DisplayFeature()
    {
        Console.WriteLine("這是一個精美的 IDE 代碼展示視窗！");
    }
}
