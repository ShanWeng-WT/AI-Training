自動化測試你的 AI

實作 Evaluation Harness 與 CI/CD 整合

為何需要自動化測試？

「沒有量化準確率，AI 永遠只是玩具。」
  • 防範回歸： 確保舊任務不崩壞
  • 量化指標： 判斷是否可上線
  • 建立信任： 降低對黑盒子的疑慮

Harness 的三大要素

  • Dataset： 輸入任務與預期結果
  • Runner： 批次執行任務
  • Evaluator： 驗證精確度與編譯結果

CI/CD 整合

「Agent 設定檔如同程式碼，需嚴格版控。」
  • 觸發測試： 發 PR 自動執行 Harness
  • 自動把關： 準確率過低阻擋合併
# 阻擋低準確率 PR
- name: Comment on PR
  if: failure()
  run: echo "❌ AI 準確率低於標準"

企業級 AI 四大守則

  • 受測試檢驗： AI 只是編譯器
  • No CI, No AI： 測試覆蓋率決定採用率
  • Prompt 版控： 所有 System Prompt 需進 Git
  • 沙盒執行： 未經同意不直接執行指令

常見陷阱與防禦機制

  • 過度優化： 需建立盲測資料集防禦
  • 裁判幻覺： 依賴確定性測試，LLM 僅輔助
