# 課程設計

* 針對從傳統開發轉換到AI開發所設計
* 以**先提供工具與技術用法 -> 再帶入概念原理以提升品質**的節奏進行
* 課程介紹的內容經過大幅簡化，排序也經過大幅客製化。 
例如不會依照Prompt Engineering -> Context Engineering -> Harness Engineering的順序，也不會一開始就從LLM的原理講起。
* 課程目標是希望能盡可能的在短時間內將講者的知識分享出來，引發拋磚引玉的效果，期待未來聽者也能給予回饋並促進交流學習
* 由於不是完全專職製作教學內容，可能會有因此疏漏或者設計不良的地方，歡迎隨時於群組詢問或私敲給予建議。。
