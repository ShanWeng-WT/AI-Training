# 講者講稿 - 第 9 堂：無痛實作大架構：Spec 驅動與 Planning Mode (PlanMode)

**預估時間:** 30 分鐘  
**目標受眾:** Unity / .NET 軟體工程師、架構師  
**主旨:** 帶領學員了解如何利用 Spec 驅動開發，結合 AI 的 PlanMode 與 Human-In-The-Loop (HITL)，安全且高效地進行大型架構重構。

---

## [00:00 - 05:00] 開場與痛點 (Intro & The Spaghetti Trap)
- **(投影片：為何需要 Spec 驅動與 PlanMode？)**
- **講者台詞：**
  「大家早安！前面我們學了 AI 如何加速寫程式，但遇到『重構 GameManager』這種大型任務時，AI 常常會吐出幾百行不能編譯、邏輯大亂的義大利麵條程式碼。你最後 debug 的時間比重寫還多。
  這堂課就是要解決這個問題。我們要讓大家的身份從『程式碼撰寫者』轉變為『架構審閱者』。」

## [05:00 - 12:00] Spec-Driven Development (撰寫精準的 Spec)
- **(投影片：撰寫精準的 Spec)**
- **講者台詞：**
  「我們要當 Tech Lead，第一步是寫出好的 Spec，框定 AI 的運算邊界。
  Spec 包含五個要素：背景、目標、非目標、限制、架構規範。
  最重要的就是『非目標 (Non-Goals)』。你要明確告訴它：不要動 UI、不要動資料庫，這能防止範圍蔓延 (Scope Creep)。」

## [12:00 - 18:00] PlanMode 展示 (PlanMode in Action)
- **(投影片：Planning Mode 核心精神)**
- **講者台詞：**
  「寫好 Spec 後，千萬別直接叫它寫程式！你要用一個咒語：『請進入 PlanMode』。
  我們要它先交出實作計畫清單、目錄異動、執行步驟。
  (展示 Prompt) 大家看，加上這句，AI 就不會亂噴 Code，而是給你一張施工藍圖。」

## [18:00 - 24:00] HITL 與架構審閱 (Human-In-The-Loop)
- **(投影片：Human-In-The-Loop (HITL) 架構審閱)**
- **講者台詞：**
  「拿到藍圖後，工程師的價值就來了。你要拿著 Checklists 檢查：有沒有耦合？有沒有效能地雷？
  像 Unity 中，事件有沒有正確 Unsubscribe？
  如果發現計畫有漏，立刻要求 AI 修正這份 Plan。改 Markdown 絕對比改 Code 輕鬆一百倍。等到計畫 100% 完美，你才回覆 Approve。」

## [24:00 - 30:00] 防禦機制與結語 (Defenses & Q&A)
- **(投影片：常見陷阱與防禦)**
- **講者台詞：**
  「最後提醒幾個陷阱。不要盲目喊 Go，最好讓它一步一步做。也不要一次把整個專案丟進去，要漸進式重構。
  掌握了 Spec 與 PlanMode，AI 就能從麻煩製造者，變成你手下最得力的打字工。
  剩下的時間開放 Q&A，謝謝大家！」
