第 4 堂：進階 Prompt Engineering 實戰：優化你的 AI 助手

🎯 課程目標 (Session Goals)

  • 回歸根本： 掌握進階「提示詞工程 (Prompt Engineering)」實戰技巧。
  • 系統化指令： 運用結構化、Few-Shot 與 Chain of Thought 等高階技巧。
  • 提升可靠度： 將模糊請求轉化為具備邏輯深度的精準約束，從源頭提升 AI 生成品質。

📖 三大核心高階技巧

不要把 Prompt 當成情緒勒索的魔法咒語，它是將意圖反編譯為約束條件的工程學：

  • 1. 結構化提示詞 (Structured Prompting)： 使用 Markdown 標籤（如 # 任務, ## 約束）精準區分資訊，減少 AI 混淆。
  • 2. 少量樣本提示 (Few-Shot Prompting)： 提供完美範例，透過模仿大幅提升輸出格式與架構的 100% 一致性。
  • 3. 思維鏈提示 (Chain of Thought, CoT)： 要求 AI「先一步一步分析邏輯，再給出程式碼」，大幅降低複雜邏輯推演的謬誤率。
💡 頓悟時刻 (Aha Moment)

進階的 Prompt Engineering 其實是在學習「如何向自己釐清問題」！

當你被迫寫下結構框架與 CoT 步驟時，你其實是在進行深度的「軟體設計前置思考」。Prompt 寫得越好，代表你的架構設計越清晰。

🛠️ 實戰演練：重構購物車折扣邏輯

❌ 階段一：直覺式的爛 Prompt

「幫我寫個計算購物車總價的 C# 函式，VIP 打 8 折，滿千折百。」

結果：寫死邏輯、缺乏異常處理與精度控制。

✔️ 階段二：加入結構化、Few-Shot 與 CoT


# 任務
實作購物車折扣計算 `DiscountCalculator`，並實作 `IDiscountCalculator` 介面。

## 約束條件
1. 金額計算必須使用 `decimal` 型別。
2. 處理參數為 null 或負數的例外狀況。

## Few-Shot 範例
### 範例：一般商品計算
```csharp
public class StandardCalculator : IDiscountCalculator {
    public decimal Calculate(decimal amount) {
        if (amount < 0) throw new ArgumentException("Amount cannot be negative");
        return amount;
    }
}
```

## 執行步驟 (Chain of Thought)
請嚴格按照以下步驟思考並寫下推論過程，最後才給出程式碼：
1. 分析傳入參數邊界條件。
2. 計算 VIP 折扣邏輯。
3. 計算滿千送百邏輯。

⚠️ 常見誤區與防禦

  • ❌ 貼上整份規格書： 導致嚴重的 Context 稀釋。
    ✔️ 防禦： 漸進式提示 (先產出 Interface，確認後再要實作)。
  • ❌ 劣質的 Few-Shot 範例： AI 會放大並複製壞味道。
    ✔️ 防禦： 範例必須是經過 Code Review 的「黃金標準」。
  • ❌ 濫用 CoT： 在簡單任務上浪費 Token 與生成時間。
    ✔️ 防禦： 純語法翻譯使用 Few-Shot 即可；複雜狀態轉換才開啟 CoT。

❓ 常見問答 (Q&A)

  • Q: 有 Copilot 補全，還需要學 Prompt 嗎？
    A: Copilot 是微觀優化；遇到宏觀設計與重構時，結構化 Prompt 才能讓 AI 成為你的「架構師副手」。
  • Q: 每次打這麼長的 Prompt 很累怎麼辦？
    A: 固化成 Agent Skills！ 辛苦寫一次完美架構，未來只需呼叫並代入變數，建立你的個人自動化工廠。
