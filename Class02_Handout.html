📖 LLM
核心概念

LLM 本質上是文字接龍引擎。它沒有「真偽」的觀念，只有「合理」的機率分佈。

理解大語言模型的三大關鍵支撐點：

  • 超級文字接龍： LLM 本質上不具備邏輯推理或程式碼編譯能力，它只是基於前文預測下一個最可能出現的 Token。
  • 以 Token 為處理單位： AI 理解代碼的最小單位不是單字或字母，而是如 =>、public 等 Token，這決定了其語義匹配特徵。
  • 沒有真實，只有合理： 當模型缺乏特定上下文時，會硬生生捏造看似語意通順但不存在的 API。這就是幻覺 (Hallucination) 的物理本質。

💡 頓悟時刻
Aha Moment

Skill 的核心科學價值在於透過「機率整形」對抗並壓制 AI 的幻覺傾向。

"Skill 的核心價值：對抗幻覺，收斂機率！"

我們在寫 Agent Skills 時，背後的數學原理是：

  • 機率整形 (Probability Shaping)： 給予精確的約束條件與 Few-shot 範例，會大幅拉高正確代碼的概率。
  • 阻斷發散： 藉由封閉邊界的定義，讓 LLM 在文字接龍時，產生偏離規範的 Token 機率被數學性地降到趨近於零。

🛠️ 實戰：
對抗幻覺

以 Unity 尋找最近敵人並扣血為例，展示缺乏約束與加入約束的產出差異。

情境： 尋找最近敵人並觸發傷害。

  • ❌ 缺乏約束： AI 隨機接龍，捏造出不存在的 Physics.FindClosestComponent API。
  • ✔️ 加入約束： 強制約束「必須使用 OverlapSphere 並透過 IDamageable 介面觸發傷害」。
// 透過 Constraints 限縮機率後，AI 穩定產生正確代碼
Collider[] hits = Physics.OverlapSphere(pos, radius);
foreach (var hit in hits) {
    if (hit.TryGetComponent<IDamageable>(out var target)) {
        target.TakeDamage(damageAmount);
    }
}

⚠️ 常見誤區與
防禦機制

生成式模型的創造力與幻覺是一體兩面的。防禦性開發思維是必不可少的護欄。

使用 LLM 開發時的三大防禦策略：

  • ❌ 依賴 AI 進行複雜計算： AI 不擅長運算或推演複雜狀態機。
    ✔️ 防禦： 讓 AI 搭建架構或寫單元測試，核心計算交由人工補足或實體編譯驗證。
  • ❌ 輕信預設的套件與 API 版本：
    ✔️ 防禦： 必須在 context 中明文指定框架版本與依賴限制（例如：C# 12 / .NET 8）。
  • ❌ 塞入過多不相干的雜訊： 導致注意力分散。
    ✔️ 防禦： 保持高訊噪比，只餵給 AI 必要且精確的專案結構與文件。

❓ 常見問答
Q&A

解惑開發團隊對大語言模型本質與限制的疑惑。

  • Q1: AI 真的理解它寫的程式碼嗎？未來會完全沒有幻覺嗎？
    A1: 不，它不理解。它基於強大的「模式匹配」捕捉關聯。而幻覺與創造力是一體兩面的，即使到 GPT-5，幻覺也無法完全根除，因此防禦性提示詞與人工審查依然至關重要。
  • Q2: 為什麼我給了 Constraints，AI 偶爾還是會犯錯（遺忘）？
    A2: 因為 AI 受到「預設立場」或上下文長度的機率權重影響。最佳防禦是配合 **Few-shot (黃金範本)**，給它看正確的範例，這比十條文字約束更有用。
