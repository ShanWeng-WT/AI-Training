## Findings

**1. Client Runtime vs Dev Tools (`FieldSetting`, `PathData`, `PathGroup`, `WaveKindXXX`)**
These are only referenced by `WaveSimulator.cs`, which is a dev-only simulation tool. The live data comes from the server at runtime. Therefore, these files have been correctly marked as `X` for the Client runtime.

**2. Generated Editor Output not in Client Data folder (`BetEnergySetting`, `FortsFireSetting`)**
These files are generated by a Unity Editor tool, but they are not saved in the client's `Data` folder (they are exported to the server). They are marked as `- (產出)` for the Unity Editor column to indicate they are produced there but reside outside the client path.

**3. Files in client `Data` with zero Unity references (`BonusWave.xml`, `NormalWave.xml`, `SpecialWave.xml`, `PathSpeedData.txt`)**
These files appear to be server-side tables that are simply co-located in the client export folder. They have zero references in the Unity project, and `PathSpeedData.txt` specifically is not used in this project at all.
