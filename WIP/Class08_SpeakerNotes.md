# 第 8 堂：制定專案的大腦：Context Files (AGENTS.md) 的威力 - 講者筆記

## ⏳ 課程時間表 (總長：30 分鐘)

### [00:00 - 05:00] 開場與痛點 (5 min)
*   **Slide: 課程目標**
*   **講者動作：** 提問：「大家是不是每次開新對話，都要重新跟 AI 說『我是用 .NET 8』、『我的命名習慣是駝峰式』？」
*   **口白：** 「今天我們要把這一切自動化。我們要為專案安裝一個『大腦』，讓 AI 一進專案就懂規矩。」

### [05:00 - 15:00] 認識 AGENTS.md & 頓悟時刻 (10 min)
*   **Slide: 為什麼需要 AGENTS.md？ & 頓悟時刻**
*   **講者動作：** 展示一個已經配有 AGENTS.md 的專案，直接下指令「新增一個 API」，展示 AI 自動套用了公司規範的架構。
*   **口白：** 「這就是頓悟時刻：我們把資深工程師的架構決策寫下來，AI 就會成為嚴格執行這些規範的代碼生成器。」

### [15:00 - 25:00] 靜態 vs 動態 Context (10 min)
*   **Slide: 實戰：靜態與動態 Context**
*   **講者動作：** 教導如何撰寫高質量的 AGENTS.md。展示 Markdown 範例。
*   **口白：** 「記住，AGENTS.md 是『靜態』的，寫死規則。不要把『動態』的商業邏輯（例如角色有多少血量）寫進去，動態邏輯讓 Agent 自己用工具去查。保持設定檔的精簡與權威性。」

### [25:00 - 30:00] 維護與總結 (5 min)
*   **Slide: 常見誤區**
*   **講者動作：** 強調 AGENTS.md 也是程式碼的一部分，需要被 Code Review。
*   **口白：** 「AGENTS.md 寫爛了，整個專案生成的 Code 都會爛。從今天起，架構變更時，第一件事就是去更新 AGENTS.md！下一堂課，我們要在這個規範下，挑戰無痛重構與 PlanMode。」
