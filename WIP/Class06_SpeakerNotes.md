# 第 6 堂：Context Engineering：精準餵養與 Token 效率最佳化 - 講者筆記

## ⏳ 課程時間表 (總長：30 分鐘)

### [00:00 - 05:00] 痛點回顧與目標 (5 min)
*   **Slide: 課程目標**
*   **講者動作：** 提問：「有沒有人貼了一大段 Code 給 AI，結果它改了 A 壞了 B，或是完全忽略你講的某句話？」
*   **口白：** 「這就是 Context 超載的問題。今天我們不談 Prompt 怎麼寫得漂亮，我們談怎麼『精準給資料』。」

### [05:00 - 15:00] Context Window 與實體隔離 (10 min)
*   **Slide: 什麼是 Context Window？ & 頓悟時刻**
*   **講者動作：** 解釋 "Lost in the Middle" (中間遺忘) 現象。
*   **口白：** 「LLM 讀文章跟人很像，開頭跟結尾記得最清楚，中間容易忘記。承接上堂課的 Subagent，我們終於迎來了頓悟：把大任務切成小任務，把大 Context 切成小 Context！前端 Agent 的 Context 裡『絕對不會』有後端的 Code，物理隔離最安全。」

### [15:00 - 25:00] 最佳化技巧實戰 (10 min)
*   **Slide: Token 效率最佳化技巧**
*   **講者動作：** 展示兩個 Prompt 差異：一個貼了整份 2000 行的檔案，另一個只貼了 50 行的 Interface 與依賴。
*   **口白：** 「不要給它實作，給它介面 (Interface)。這不僅省 Token 幫公司省錢，更能大幅提高 AI 生成的準確率。」

### [25:00 - 30:00] Q&A 與下堂預告 (5 min)
*   **Slide: 常見誤區**
*   **講者動作：** 總結「少即是多」的概念。
*   **口白：** 「定期清理對話（Clear Context）是好習慣。前兩堂課我們講了 Subagents 跟 Context，下一堂課我們要把這一切串起來，看看 Agent 的底層到底是怎麼運作的。」
