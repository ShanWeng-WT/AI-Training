# 第 7 堂：揭開 LLM 的黑盒子：它能做什麼，不能做什麼？
## 講者講稿 (Speaker Notes)

**課程總長：** 30 分鐘
**目標受眾：** 研發團隊全體成員
**設備需求：** 投影機、白板

---

### [00:00 - 05:00] 開場與回顧 (Intro & Hook)
**(Slide 1: 課程目標)**

**講者：**
「各位好，歡迎回到第七堂課。在前面的課程中，我們已經實作了各種 Agent 技巧、分工協作與 Context 優化。但要真正馴服這匹猛獸，我們必須理解它的『物理極限』。

今天，我們要揭開 LLM 的黑盒子，了解它為什麼會胡說八道，以及我們如何利用數學與機率來操控它。」

---

### [05:00 - 15:00] LLM 的底層邏輯 (Core Concepts)
**(Slide 2: LLM 核心概念解析)**

**講者：**
「大家一定覺得 AI 好像什麼都懂。但這是一個危險的錯覺。

本質上，LLM 只是一個超級強大的『文字接龍』引擎。它不知道什麼是 C#，只知道在特定情況下，預測下一個 Token 的機率。

最可怕的是，它沒有『真實』的概念，只有『合理』的概念。當遇到不懂的東西，它會硬生生捏造一段看起來語法全對、但呼叫了不存在 API 的程式碼。這就是所謂的幻覺（Hallucination）。」

**(Slide 3: 頓悟時刻 Aha Moment)**

**講者：**
「理解了這一點，我們就能更深層地理解之前學過的 Agent Skills 的本質。Skill 到底是什麼？

Skill 就是用來『對抗幻覺』的工具！我們加入的約束和範例，本質上是在做『機率整形』。當你給它一個完美的 C# 範例，它預測出偏離規範的 Token 機率就會被數學性地大幅壓低。」

---

### [15:00 - 25:00] 幻覺實例與破解 (Live Example)
**(Slide 4: 實戰演練：對抗幻覺的策略)**

**講者：**
「來看看這個 Unity 的例子。如果你叫 AI 找最近的敵人並扣血，它可能會發明出 `Physics.FindClosestComponent` 這種根本不存在的 API。因為在它的開源訓練數據裡，這種命名太常見了。

要破解它，我們必須在 Prompt 或 Skill 裡強制寫入 Context：『必須使用 OverlapSphere，必須透過 IDamageable 介面』。透過這種限縮機率的方式，我們成功阻斷了它向外發散捏造 API 的可能。」

---

### [25:00 - 30:00] 誤區與總結 (Wrap up & Q&A)
**(Slide 5: 常見誤區與防禦 & Slide 6: 常見問答 Q&A)**

**講者：**
「我們最常犯的錯，就是把它當成計算機。LLM 算數學或推演複雜狀態機容易出錯。遇到複雜邏輯，請讓它寫架構，核心計算你自己補，或是請它寫單元測試來驗證。

另外，永遠記得指定你的套件版本，並且保持給予它的 Context 乾淨、精確。

有人會問：未來的 GPT-5 出來，幻覺會消失嗎？答案是：不會完全消失。生成式模型的創造力與幻覺是一體兩面的。這也是為什麼我們作為工程師，防禦性提示詞與人工審查永遠不可或缺。

今天先到這邊，下一堂課（第 8 堂）我們會教大家如何建立整個專案的大腦：AGENTS.md。謝謝大家！」
