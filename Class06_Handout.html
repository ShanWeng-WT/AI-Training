第 6 堂：Context Engineering

精準餵養與 Token 效率最佳化

🎯 課程目標

  • 掌握 Context Window： 理解 AI 記憶力的極限。
  • 實體隔離： 學習為不同 Agent 準備專屬資料。
  • 提升命中率： 減少雜訊，提升程式碼生成的精準度。

📖 什麼是 Context Window？

AI 的「工作記憶」是有上限的（Token 限制）。

  • 餵太多： AI 會「失憶」或「失焦」，忽略重要資訊（Lost in the Middle 現象）。
  • 餵太少： AI 只能靠瞎猜（產生幻覺）。
  • 關鍵： 只給剛剛好的 Context。
💡 頓悟時刻 (Aha Moment)

Subagents 最大的優勢是能對 Context 進行「實體隔離」！前端 Agent 絕對不會看到後端資料，徹底避免領域資訊互相干擾。

🛠️ Token 效率最佳化技巧

  • 去蕪存菁： 不要直接丟整個專案，只給相關的介面 (Interface) 或依賴檔。
  • 結構化輸入： 使用 Markdown, JSON 或 XML 標籤讓 AI 易於解析。
  • 動態提取： 透過 RAG 或工具指令 (如 grep) 即時取得需要的程式碼片段。

⚠️ 常見誤區

  • 一鍵梭哈 (Dump all)： 把十萬行程式碼全部塞進 Prompt。
  • 歷史包袱： 沒有定期清理對話紀錄，導致舊 Context 影響新判斷。
