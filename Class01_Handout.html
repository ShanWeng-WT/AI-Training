Agent Skills 的定義與運用

Skill 是什麼？

將不可控的隨機對話，升級為品質穩定的套裝「技能」輸出。

😰 無技能的痛點

  • 重新探索成本每次執行相同任務，AI 都需重新推演解法，耗費額外時間與 Token。
  • 缺乏一致的標準AI每次解法與結論可能截然不同。
  • 品質難以掌握模型發散的輸出，導致難以預期與控管最終品質。

💡 核心解決方案

  • 固化專案知識在 SKILL.md 中載明團隊固定的開發知識、指令與標準作法。
  • 可執行的工作流透過內建的程式腳本、明定的指令，讓 AI 按既定邏輯執行，確保產出一致。
  • 賦予行為邊界不再依賴單次對話，改用系統化定義來約束 AI 行為。

🎯 達成的目標

  • 穩定且精準產出結果變得可預測且高度一致。
  • 經驗資產化將踩坑經驗與最佳實踐轉化為隨插即用的「技能」。
  • 提升效益比杜絕反覆溝通與無效試錯，節省時間及Token。

Agent Skill 的 標準與結構

每一個 Skill 都必須遵循特定的目錄結構與命名規範，才能被系統正確載入並辨識。

  • 📁 上層目錄：Skill 的根目錄視工具而異。最常見為 .agents/skills/，也有.claude/skills或.github/skills 。
  • 🏷️ 命名規範：目錄名稱與 name 屬性必須完全一致，建議使用 kebab-case (小寫與連字號，如 my-skill)。
  • 📄 SKILL.md (必填)：技能的核心定義檔，上方必須包含 YAML Frontmatter 宣告。
  • 📂 其他目錄 (選填)：如 scripts/ 放置腳本、references/ 放置參考文件、assets/ 放置模板檔案。
Directory & SKILL.md

.agents/skills/             # Depends on the harness/tool
└── your-skill-name/ 
	├── SKILL.md        # Required - main skill file 
	├── scripts/        # Optional - executable code 
	├── references/     # Optional - documentation 
	└── assets/         # Optional - templates, etc.

# SKILL.md 檔案內容開頭必須包含 YAML Frontmatter：
---
name: your-skill-name
description: What it does. Use when user asks to [specific phrases].
---

Skill 的範例

以 fish-data-usage-analysis 技能為例，一個結構完整的 Skill 通常會明確定義以下核心要素：

  • 🎯 明確目標 (What it produces)具體定義 AI 任務完成後應產出的結果與格式。
  • ⚖️ 規則與約束 (Symbol rules)定義判斷標準，提供統一的邏輯準則，避免 AI 產生幻覺或隨意猜測。
  • 👣 執行步驟 (Steps)將複雜任務拆解為可逐步驗證的工作流，引導 AI 按部就班執行。
SKILL.md 節錄
# Data Usage Analysis (Server / Client / Unity Editor)

## What this produces
A markdown table classifying every exported data table by where it is actually **referenced at runtime**...

## Symbol rules (apply consistently)
| Symbol | Meaning |
|--------|---------|
| **O**  | Confirmed referenced at runtime... |
| **X**  | Confirmed unused... |

## Step 1 — Resolve the roots
pwsh -File <skill>/scripts/Resolve-Roots.ps1 ...

## Step 2 — Build the data-entity inventory
...
## Step 3 — Dispatch two analysis subagents in parallel
Run a **server** subagent and a **client** subagent in the same turn...

如何使用 Skill?

三種喚醒方式

🗣️ 自然語言對話
「請對 fish 專案資料表進行分析」
若 Agent 判斷需求符合 Skill 描述，就會自動發動。
🎯 指定檔案
「使用 fish-data-usage-analysis 幫我分析...」
明確告知使用的技能與目標。
⚡ 快捷指令喚醒
輸入 @ 或 /skill
在 IDE/CLI 快速引用並帶入上下文。

延遲載入機制

為了兼顧效能與精準度，避免上下文雜訊：

  • 1️⃣ 不自動載入所有內容
    對話開始時只載入名稱(Name)與描述 (Description)，做為判斷依據，節省 Token。
  • 2️⃣ 意圖比對觸發
    當需求符合描述，或使用快捷指令，即判定觸發。
  • 3️⃣ 完整載入執行
    此時才讀取完整 SKILL.md，嚴格遵循步驟與規範執行任務。

如何自製 Skill?

製作專屬技能其實很簡單，你可以依據情境選擇以下四種方式：

  • 1. 使用現有 Skill 輔助生成
    利用 skill-creator 或 writing-skills 等輔助技能來幫你產出。
  • 2. 直接和 AI 描述需求
    直接用自然語言說明流程，讓 AI 自動轉換成 SKILL.md 格式。
  • 3. 成功任務後順手固化
    在完成一次滿意的任務後，直接指示 AI：「把剛才的流程做成一個 Skill」。
  • 4. 從頭到尾自己寫
    自行建立目錄與定義檔，精確控制腳本與複雜的約束條件。
