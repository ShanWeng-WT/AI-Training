打造專屬助手：Agent Skills 的定義與運用

🎯 課程目標 (Session Goals)

  • 建立技能： 實務操作 Agent Skills
  • 翻轉模式： 從對話轉為系統化技能
  • 解決痛點： 降低除錯成本，生產力大躍進

📖 背景與核心痛點

  • 通用 AI 局限： 不懂專案架構與規範
  • 同步成本： 反覆糾錯，拖慢效率
  • 解法： Agent Skills 賦予專案感知力

⚙️ Agent Skills 的核心要素

  • Intent： 觸發條件
  • Context： 專案上下文
  • Constraints： 絕對不能做什麼
  • Few-shot： 黃金範本
  • Next Actions： 任務後續流程

💡 頓悟時刻 (Aha Moment)

Skill 不是 Prompt！它是「可重複使用的自動化工作流」。
  • 知識軟體化
  • 經驗傳承
  • 提升代碼品質

🛠️ 實戰：.NET Repository Generator

  • 痛點： AI 直接在 Controller 操作 DbContext
  • 解法： 1. 系統提示詞 2. 黃金範本

# Constraints (約束條件):
1. 禁止在 Controller 直接用 DbContext
2. 建立 `I[實體]Repository` 介面
3. 必須使用 `async Task<T>`

⚠️ 常見誤區與防禦機制

  • 空泛指令 → 具體條件 (如：低於 50 行)
  • 缺乏上下文 → 限制範圍 (指定介面/框架)
  • 無防呆機制 → 自我修正 (要求 AI 自我檢查)

❓ 常見問答 (Q&A)

  • 拖慢進度？ → 是投資！ (30分換無數除錯時間)
  • Token 爆掉？ → 動態上下文！ (限縮範圍降成本)
