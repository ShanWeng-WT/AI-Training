第 1 堂：打造專屬助手：Agent Skills 的定義與運用

🎯 課程目標 (Session Goals)

  • 從零開始建立 Agent Skills： 引導 Unity 前端與 .NET 後端開發人員實務操作。
  • 翻轉傳統 AI 互動模式： 將發散式對話轉化為系統化的 AI 技能。
  • 解決企業真實痛點： 透過具體約束與範例，降低溝通與除錯成本，實現生產力跳躍式提升。

📖 背景與核心痛點

  • 通用 AI 的局限性： 通用解法往往不符合專案架構（如：Unity ECS 架構、.NET Repository 規範）。
  • 反覆同步上下文的成本： 不斷糾正錯誤，降低了開發效率。
  • 解決方案：Agent Skills： 賦予 AI 特定目標與專案感知能力。

⚙️ Agent Skills 的核心要素

  • 觸發條件 (Intent)： 什麼時候該使用？明確任務意圖。
  • 上下文 (Context)： 專案規範與特定套件清單。
  • 約束條件 (Constraints)： AI 的防呆機制，明確告知絕對不能做什麼。
  • 範例 (Few-shot)： 提供黃金範本，讓 AI 精準模仿。
  • 後續行動 (Next Actions)： 任務完成後的自動化流程（如執行單元測試）。
💡 頓悟時刻 (Aha Moment)

Skill 不是 Prompt！ 它是「可重複使用的自動化工作流」。

將大腦中的領域知識 (Domain Knowledge) 軟體化，讓資深工程師的經驗能透過 AI 傳承，大幅提升團隊代碼品質！

🛠️ 實戰演練：.NET Repository Generator

痛點： AI 直接在 Controller 操作 DbContext。

1. 定義系統提示詞 (System Prompt)


# Skill: .NET Repository Generator
**Constraints (約束條件):**
1. 絕對禁止在 Controller 層直接使用 DbContext。
2. 必須建立 `I[實體名稱]Repository` 介面。
3. 所有方法必須使用非同步 `async Task<T>`。

2. 加入黃金範本 (Few-shot)


// 範例：完美的 Repository 實作
public class UserRepository : IUserRepository {
    // 實作程式碼與例外處理...
}

⚠️ 常見誤區與防禦機制

  • ❌ 指令過於空泛： 「幫我寫好維護的程式」 → ✔️ 具體條件： 「方法長度低於 50 行」。
  • ❌ 缺乏上下文： AI 發明不存在的類別 → ✔️ 限制範圍： 指定要讀取的介面與專案框架。
  • ❌ 無防呆機制： AI 產出錯誤程式碼 → ✔️ 自我修正： 加入「請自我檢查是否違反約束並重寫」。

❓ 常見問答 (Q&A)

  • Q: 撰寫 Skill 會拖慢進度嗎？
    A: 這是一種投資！初期花 30 分鐘，未來每位成員都能省下無數次除錯與 Code Review 時間。
  • Q: 專案太大，AI Token 會爆掉嗎？
    A: 透過動態上下文（如 @Files），限縮 AI 讀取的範圍，降低幻覺與成本。
